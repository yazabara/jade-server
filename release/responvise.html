Отзывчивый веб-дизайн

  • Responsive web design

  • отзывчивый вед-дизайн

Отзывчивый дизайн зачем?

  • Интеллектуальная автоматическая подстройка контента и навигации под все модели мобильных устройств;

  • Отсутствие статичности. Это значит, что верстка ресурса выполнена не под определенное разрешение экрана, при котором более мелкие или крупные экраны будут отображать информацию некорректно, а будет гармонично подстраиваться под любой экран;

  • Нет необходимости создавать несколько различных версий сайта, которые подходили бы для планшетов, обычных персональных компьютеров, смартфонов и мобильных телефонов;

  • Не существует вероятности возникновения проблем с редиректом, которым подвержены мобильные версии.

Этапы делаем по-очереди

  • Создаем прототипы страниц для различных разрешений экранов;

  • Разрабатываем дизайн, с учем всех перемещений блоков, в зависимости от расширения экрана;

  • Верстаем адаптивный дизайн сайта (это один из наиболее важных этапов работы);

  • Тестирование, perfectpixel

  • Дальше интегрируем, если это какой-либо сервис, а не статический сайт

Внимание

Приход адаптивных сайтов полностью изменил процесс разработки. Если раньше макеты дизайна сдавались как этап и уходили дальше по производственной цепочке, то для создания responsive сайта необходимо плотное взаимодействие дизайнера и front-end разработчика на большинстве этапов разработки.В зависимости от масштабов компании роли проектировщика, front-end разработчика и дизайнера могут быть смешаны.
